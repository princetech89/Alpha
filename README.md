## 🌟 What is AlphaSignal?

</div>

> **AlphaSignal** is a **full-stack AI trading intelligence platform** that detects **28 technical chart patterns** across NSE & BSE stocks in real time. It combines live market data from **Angel One SmartAPI** and **Upstox** with **OpenAI GPT-4o-mini** and **Pinecone RAG** to deliver pattern signals with confidence scores, plain-English AI explanations, historical walk-forward back-tested success rates, and an intelligent stock assistant.

## 🚀 Getting Started

### ✅ Prerequisites

| Requirement | Version | Notes |
|---|:---:|---|
| **Node.js** | 20+ | LTS recommended |
| **npm** | 8+ | Comes with Node.js |
| **PostgreSQL / Neon** | Latest | [Neon](https://neon.tech) free tier works great |
| **Angel One SmartAPI** | — | Requires brokerage account |
| **Upstox API** | — | Developer account needed |
| **OpenAI API** | — | GPT-4o-mini + embedding access |
| **Pinecone** | — | Serverless index on free tier |

---

### Step 1 — Clone and Install

```bash
# Clone the repository
git clone https://github.com/princetech89/Alpha.git

# Navigate into the project
cd Alpha

# Install all dependencies
npm install
```

---

### Step 2 — Configure Environment Variables

Create a `.env` file in the project root with the following variables:

```env
# ── Angel One SmartAPI ──────────────────────────────────────────────────────
ANGEL_API_KEY=your_angel_api_key
ANGEL_CLIENT_ID=your_client_id
ANGEL_PASSWORD=your_login_password
ANGEL_TOTP_SECRET=your_totp_secret

# ── Upstox ──────────────────────────────────────────────────────────────────
UPSTOX_CLIENT_ID=your_upstox_client_id
UPSTOX_CLIENT_SECRET=your_upstox_client_secret
UPSTOX_REDIRECT_URI=http://localhost:5005/upstox-callback
UPSTOX_ANALYTICS_TOKEN=your_upstox_analytics_jwt_token

# ── OpenAI ──────────────────────────────────────────────────────────────────
OPENAI_API_KEY=sk-...

# ── Pinecone (AlphaSignal Bot knowledge base) ───────────────────────────────
PINECONE_API_KEY=pcsk_...
PINECONE_INDEX=alphasignal-kb

# ── Database ────────────────────────────────────────────────────────────────
DATABASE_URL=postgresql://user:pass@host/db?sslmode=require

# ── Server ──────────────────────────────────────────────────────────────────
PORT=5005
SESSION_SECRET=your_random_secret_string_min_32_chars
NODE_ENV=development
```

> ⚠️ **Never commit your `.env` file.** It is already in `.gitignore`.

---

### Step 3 — Push Database Schema

```bash
npm run db:push
```

This creates the `profileUsers` table in your PostgreSQL database via Drizzle ORM.

---

### Step 4 — (Optional) Ingest Knowledge Base Documents

Place PDF files in the `docs/` folder, then run:

```bash
node scripts/ingest-kb.mjs
```

This chunks and indexes the PDFs into Pinecone so the **AlphaSignal Bot** can answer questions from them using RAG.

---

### Step 5 — Start the Development Server

```bash
npm run dev
```

Open **[http://localhost:5005](http://localhost:5005)** in your browser.

The development server serves both the Express API and Vite HMR frontend on the same port.

## ⚙️ How Pattern Detection Works

```
┌─────────────────────────────────────────────────────────────────────┐
│                     AlphaSignal Detection Pipeline                  │
└─────────────────────────────────────────────────────────────────────┘

  1. QUOTE FETCH              2. HISTORICAL CANDLES        3. INDICATORS
  ┌──────────────┐           ┌──────────────────┐       ┌─────────────────┐
  │ Angel One    │ ────────► │  120 days daily  │ ────► │ RSI, MACD, BB   │
  │ SmartAPI     │           │  (Upstox backup) │       │ SMA-20/50, ATR  │
  └──────────────┘           └──────────────────┘       └────────┬────────┘
                                                                  │
  4. PATTERN DETECTION        5. BACKTESTING           6. AI ENRICHMENT
  ┌──────────────┐           ┌──────────────────┐       ┌─────────────────┐
  │ 28 Detectors │ ────────► │ Walk-Forward BT  │ ────► │ GPT-4o-mini     │
  │ run in TS    │           │ on stock history │       │ Entry/Stop/Target│
  └──────────────┘           └──────────────────┘       └────────┬────────┘
                                                                  │
                                                       7. CACHED RESPONSE
                                                       ┌─────────────────┐
                                                       │ 5-minute cache  │
                                                       │ SSE streaming   │
                                                       └─────────────────┘
```

**Detection Pipeline Explained:**

1. 🔌 **Quote Fetch** — Angel One SmartAPI batch-fetches live FULL quotes (up to 50 symbols/call) for all NSE/BSE symbols at scan time
2. 📅 **Historical Candles** — 120 days of daily OHLCV candles from Angel One (Upstox fallback if Angel One fails)
3. 🧮 **Indicator Computation** — TypeScript `calculateIndicators()` computes RSI, MACD, Bollinger Bands, SMA-20, SMA-50, ATR, and volume metrics
4. 🔍 **28 Detectors Run** — Each pattern function evaluates candle + indicator data and returns a `Signal` (or `null`)
5. 📈 **Walk-Forward Backtest** — `backtestPattern()` scans the stock's own 120-day history to compute a real historical success rate per pattern
6. 🤖 **AI Enrichment** — OpenAI GPT-4o-mini generates a plain-English explanation with entry/stop/target guidance
7. ⚡ **Cache** — Results are cached for 5 minutes. The server pre-warms both NSE & BSE caches on startup

### 🏗️ How Vercel Deployment Works

The `npm run build:vercel` script:
1. **Vite builds** the React frontend to `dist/public/`
2. **esbuild bundles** the entire Express server (`api/index.ts` + all of `server/`) into a single `api/bundle.js` serverless function

All API routes and the Upstox OAuth callback are routed through `api/bundle.js` via `vercel.json` rewrites. The React SPA is served from `dist/public/`.
